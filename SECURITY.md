# Security Policy

NexLog is a local-first DFIR tool, so security reports may involve sensitive
case data, evidence handling, or analyst workstations.

Please do not publish exploit details or real evidence samples in public issues.
Use GitHub private vulnerability reporting when available, or contact the
maintainer privately with:

- Affected version or commit.
- Reproduction steps.
- Impact and affected component.
- Suggested fix, if known.

Supported versions and additional handling guidance are documented in
[docs/security.md](docs/security.md).
